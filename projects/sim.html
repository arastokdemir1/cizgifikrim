Office Tycoon

Office Tycoon, tarayıcıda çalışan küçük bir simülasyon fikrinin ötesinde; API, web ve desktop yüzlerini aynı workspace mantığında taşıyan daha geniş bir ürün yapısı. `office-tycoon-pro` içindeki monorepo yaklaşımı, simülasyon çekirdeğini birden fazla istemciye yaymak için düşünülmüş.

Workspaces apps/* Model
API Service Surface
Web Interactive UI
Desktop Packaged Runtime
// ÜRÜN MANTIĞI

Neden yalnızca oyun değil?

Kök `package.json`, `apps/*` workspace modelini açıkça tanımlıyor ve aynı ürün içinde `api`, `web` ve `desktop` yüzlerinin birlikte geliştirildiğini gösteriyor. Bu, projenin yalnızca browser tabında kalan küçük oyun denemesi olmadığını; ekonomik mantık katmanını farklı istemci biçimlerinde yaşatmak istediğini kanıtlıyor. Simülasyon burada yalnızca eğlence değil, ürün mimarisi için de bir test alanı.

Tek Çekirdek, Çok Yüz

Aynı mantık katmanının API, web ve desktop dağıtımına hazırlanması, projeyi genişleyebilir hale getiriyor.

Dağıtım Düşüncesi

Desktop package komutlarından önce web hazırlama scriptleri çalıştırılıyor; bu da ürünleşme niyetini açıkça gösteriyor.

Uzun Vadeli Alan

Monorepo yaklaşımı, simülasyon çekirdeğinin zamanla farklı modüllerle büyümesine uygun bir zemin hazırlıyor.

// ÇALIŞMA AKIŞI

Geliştirme modeli nasıl kurgulanmış?

Kök komutlar, `concurrently` kullanarak farklı çalışma alanlarını aynı anda ayağa kaldırıyor. Bu yaklaşım, simülasyon mantığını tek tarafta dondurmak yerine ürün yüzlerini birlikte geliştirmeyi kolaylaştırıyor. Özellikle desktop paketi öncesi web build alınması, kullanıcıya ulaşacak son deneyimin yalnızca geliştirme moduna bırakılmadığını gösteriyor.

[ROOT] workspace package.json okunuyor [DEV] api + web eşzamanlı ayağa kalkıyor [DESKTOP] isteğe bağlı desktop runtime üçüncü kanal olarak ekleniyor [BUILD] desktop packaging öncesi web derleniyor [DISTRIBUTION] aynı simülasyon çekirdeği farklı yüzlere taşınıyor
// DEĞER

Bu projeyi neden ciddiye almak gerekir?

Birçok simülasyon denemesi yalnızca fikir aşamasında kalır. Office Tycoon’un farkı, bunun baştan çok yüzlü ürün mantığıyla ele alınması. Bu sayede proje, basit web oyunu yerine gelecekte masaüstüne ve servis katmanına açılabilecek bir simülasyon platformu gibi düşünülebilir.
