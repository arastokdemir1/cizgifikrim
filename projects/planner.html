LOCAL-FIRST PRODUCTIVITY

Aether

Aether, web teknolojilerini masaüstü kabuğuna sarmalayan sıradan bir Electron denemesi değil; tek örnek kilidi, yerel kullanıcı bootstrap’i, SQLite veri tabanı, bildirim kanalı, cron akışı ve macOS hardened runtime ile ürünleşmiş local-first planlama sistemi kurmaya çalışan ciddi bir masaüstü çekirdeği.

Electron Desktop Shell
SQLite Local Data
Socket.IO Realtime Alerts
Hardened Mac Runtime
// ÜRÜN ODAĞI

Aether neyi hedefliyor?

Birçok planlayıcı bulut merkezli çalışır; Aether ise yerel veri hâkimiyetini temel alan masaüstü ürününe daha yakın duruyor. `main.js` Electron pencere yaşam döngüsünü yönetirken, `server.js` tarafı local kullanıcıyı garanti altına alıyor, eski JSON veriyi SQLite tarafına taşıyan migrasyon çalıştırıyor ve tüm görev akışını yerel kimlik üzerinden birleştiriyor. Bu nedenle proje, takvim/görev arayüzünden daha fazlası olarak düşünülmüş.

Yerel Kimlik

`local-desktop-user` yaklaşımı, çok kullanıcılı bulut yapısı yerine masaüstü sahibini merkeze alan local-first model kuruyor.

Migrasyon Disiplini

Eski `db.json` verisinin SQLite şemasına taşınması, projenin prototipten ürünleşmeye ilerlediğini gösteriyor.

Paketleme Ciddiyeti

Universal DMG, entitlements ve hardened runtime ayarları, Aether’in gerçekten dağıtılan bir masaüstü ürününe dönüştürüldüğünü gösteriyor.

// ÇEKİRDEK AKIŞ

Uygulama arkada nasıl işliyor?

Electron penceresi açılmadan önce Express tabanlı yerel sunucu ayağa kalkıyor. API sağlık kontrolü, statik public yüz, görev normalize etme, renk/öncelik eşleme ve Socket.IO bildirim hattı aynı servis içinde toplanıyor. Böylece Aether yalnızca HTML paketleyen Electron uygulaması olmuyor; kendi backend’i ile yaşayan masaüstü ürünü haline geliyor.

[BOOT] local server başlatılıyor [LOCK] ikinci instance engelleniyor [DB] local kullanıcı ve migration kontrolü çalışıyor [API] görev / alışkanlık uçları hazır hale geliyor [WINDOW] Electron yalnızca server hazır olunca yükleniyor
// STRATEJİK GÜÇ

Projeyi değerli yapan yön

Aether’in güçlü tarafı, verimlilik uygulamasını “sadece UI” olarak görmemesi. Veri modeli, migration, bildirim, paketleme ve işletim sistemi sertliği birlikte ele alınıyor. Bu yüzden proje, masaüstü planlayıcı pazarında yerel veri kontrolünü önemseyen ciddi bir alternatif olabilecek kadar sağlam bir temel sunuyor.
